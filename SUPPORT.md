## Bug reports

Open a [GitHub Issue](../../issues/new/choose) using the **Bug report** template. Please include reproduction steps, expected vs. actual behavior, your environment (OS, Node version, browser), and any relevant logs.

## Feature requests

Open a [GitHub Issue](../../issues/new/choose) using the **Feature request** template. Describe the use case first — "I want to be able to X so that Y" — before jumping to a proposed solution.

## Open-ended questions

For questions that aren't a bug or a feature request — design discussions, "how would I…" questions, sharing what you built on top — open a [GitHub Issue](../../issues/new/choose) and pick the template that fits closest (use **Feature request** for proposals). Describe what you're trying to do and we'll triage from there.

## Security issues

Please **don't** open a public issue for security vulnerabilities. See [`SECURITY.md`](./SECURITY.md) for the responsible-disclosure process.

## Commercial / hosting questions

The hosted version at [price.games](https://price.games) is run by the original maintainer. For questions about that specific deployment (account issues, content concerns, partnership inquiries), use the contact form on the live site rather than the issue tracker.

## Response times

This is a community project with no SLA. Maintainers respond when they can. Bug reports with clean reproductions get triaged fastest.
